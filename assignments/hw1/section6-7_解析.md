## 📋 第六部分概述：轨迹实现

### 6.1 轨迹节点概述

**文件位置**：`assignment1/src/trajectory.cpp`

**节点功能**：
- 发布与 GUI 相同的消息类型
- 实现自动轨迹生成（替代手动 GUI 控制）

**⚠️ 关键注意事项**：
- **使用轨迹生成器时，必须关闭 GUI**
- 点击 GUI 的红色按钮（Stop/Disable）
- 否则 GUI 会持续发送命令，与轨迹节点冲突

### 6.2 工作流程

**数据流**：
```
用户发布目标点 → /setpoint 话题 → trajectory 节点订阅
    ↓
trajectory 节点计算轨迹
    ↓
发布关节命令 → /ur5e_controller/joint_trajectory
    ↓
机器人执行运动
```

### 6.3 发布目标点

**命令格式**：
```bash
$ ros2 topic pub -1 /setpoint geometry_msgs/msg/Point '{x: 0.5, y: 0.2, z: 0.5}'
```

**参数解释**：
- `-1`：发布一次后退出（`-1` 表示发布1次）
- `/setpoint`：话题名称
- `geometry_msgs/msg/Point`：消息类型（3D点）
- `{x: 0.5, y: 0.2, z: 0.5}`：目标坐标（米）

**⚠️ 重要提示**：
- 示例坐标 `(0.5, 0.2, 0.5)` 仅用于演示
- **必须根据机器人当前位置调整坐标**
- 确保目标点在机器人工作空间内

### 6.4 轨迹节点功能

**主要功能**：
1. **订阅** `/setpoint` 话题（接收目标点）
2. **计算** 从当前位置到目标点的轨迹
3. **插值** 夹爪位置（平滑移动）
4. **发布** 关节轨迹命令

**实现方式**：
- 使用 Resolved Motion Rate Control（第5部分介绍的方法）
- 逐步移动，直到到达目标点

## 💻 6.5 需要实现的函数

### 文件位置

**主要实现文件**：`assignment1.cpp`

**查找方法**：在文件中搜索 `TODO` 注释

### 函数列表（共9分）

#### 1. ForwardKinematics（3分）

**功能**：计算夹爪的当前位置

**输入**：
- 关节角度：$q_1, q_2, q_3, q_4, q_5, q_6$

**输出**：
- 4×4 齐次变换矩阵
- 表示 `gripper_pick` 相对于 `world` 的位置和姿态

**实现要求**：
- 填充 4×4 矩阵的所有16个元素
- 不要假设任何元素为0
- 必须显式设置每个元素

**数学形式**：
$$T_{gripper}^{world} = \begin{bmatrix} R & t \\ 0 & 1 \end{bmatrix}$$

#### 2. ForwardKinematicsInverse（1分）

**功能**：计算前向运动学的逆

**输入**：
- 4×4 齐次变换矩阵 $T$

**输出**：
- 4×4 逆变换矩阵 $T^{-1}$

**用途**：
- 计算目标点在 gripper 坐标系中的坐标
- 用于计算误差向量 $e$

**数学公式**：
$$T^{-1} = \begin{bmatrix} R^T & -R^T t \\ 0 & 1 \end{bmatrix}$$

#### 3. AdjointTransformationInverse（2分）

**功能**：将空间速度转换为 body 速度

**输入**：
- 旋转矩阵 $R$（3×3）
- 平移向量 $t$（3×1）

**输出**：
- 6×6 逆伴随变换矩阵 $Ad^{-1}$

**数学公式**：
$$Ad^{-1} = \begin{bmatrix} R^T & -R^T \hat{t} \\ 0 & R^T \end{bmatrix}$$

其中 $\hat{t}$ 是平移向量 $t$ 的反对称矩阵。

#### 4. Jacobian（3分）

**功能**：计算操作臂空间雅可比矩阵

**输入**：
- 关节角度：$q_1, q_2, q_3, q_4, q_5, q_6$

**输出**：
- 6×6 空间雅可比矩阵 $J(q)$

**矩阵结构**：
$$J(q) = \begin{bmatrix} J_v(q) \\ J_\omega(q) \end{bmatrix}$$

其中：
- $J_v(q)$：3×6 线速度雅可比（前3行）
- $J_\omega(q)$：3×6 角速度雅可比（后3行）

## ⚠️ 6.6 关键警告

### 矩阵初始化警告

**⚠️ 必须注意**：

> **"Be careful to fill in all the elements of these matrices, do not assume that all values are initially set to '0'. I repeat: make sure you set all the elements of the all the matrices in your code."**

**含义**：
- ❌ **不要假设**矩阵元素初始化为0
- ✅ **必须显式设置**每个矩阵元素
- ✅ 确保所有16个元素（4×4矩阵）或36个元素（6×6矩阵）都被设置

**为什么重要？**
- 未初始化的值可能是随机数
- 会导致计算结果错误
- 难以调试（错误不明显）

**正确做法**：
```cpp
// 错误示例（不要这样做）
Matrix4d T;
T(0,0) = 1.0;  // 只设置了部分元素
// 其他元素可能是未定义的！

// 正确示例
Matrix4d T = Matrix4d::Zero();  // 先初始化为0
T(0,0) = 1.0;  // 然后设置所有元素
T(0,1) = 0.0;
// ... 设置所有16个元素
```

## 💡 6.8 实现提示（Hints）

### 提示1：从小动作开始

**策略**：
- ✅ 从已知的良好初始配置开始
- ✅ 先实现小的移动（例如10cm）
- ✅ 验证小动作正确后再尝试大动作
- ❌ 不要一开始就尝试复杂的运动

**原因**：
- 小动作更容易调试
- 容易发现错误
- 避免机器人进入奇异配置

### 提示2：非奇异初始配置

**什么是奇异配置？**
- 雅可比矩阵的行列式为0（或接近0）
- 无法唯一求解逆运动学
- 机器人失去某些方向的运动能力

**推荐的初始配置**：

| 关节名称 | 角度（弧度） | 角度（度） |
|---------|------------|-----------|
| `shoulder_pan_joint` | 0.0 | 0° |
| `shoulder_lift_joint` | -1.5708 | -90° |
| `elbow_joint` | -1.5708 | -90° |
| `wrist_1_joint` | -1.5708 | -90° |
| `wrist_2_joint` | 1.5708 | 90° |
| `wrist_3_joint` | 0.0 | 0° |

**使用建议**：
- 在尝试其他配置之前，先使用这个配置
- 这个配置让机器人"直立"，远离奇异点
- 适合测试和调试

**如何设置**：
1. 使用 GUI 滑块设置这些角度
2. 或使用 `ros2 topic pub` 发布关节位置

### 提示3：处理奇异配置

**问题**：
- 机器人在轨迹中可能遇到奇异配置
- 终端会打印警告消息

**预防措施**：
- ✅ 从良好的非奇异初始配置开始
- ✅ 执行相对小的运动（约10cm）
- ✅ 避免大范围移动

**如果遇到奇异配置**：
1. 机器人会停止移动
2. **关闭所有ROS进程**
3. 重新启动仿真
4. 使用不同的初始配置或目标点

### 提示4：验证结果

**使用 tf2_echo 验证**：

```bash
$ ros2 run tf2_ros tf2_echo world gripper_pick
```

**检查项**：
- ✅ 夹爪是否到达目标坐标
- ✅ 位置误差是否在可接受范围内（< 1cm）
- ✅ 方向是否保持不变（如果要求）

**验证步骤**：
1. 发布目标点
2. 等待机器人移动完成
3. 使用 `tf2_echo` 查看实际位置
