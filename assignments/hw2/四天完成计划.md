## 任务解析（663 版：在做什么？）

### 1. 实际场景

你有一个**机械臂**，末端装了一个**摄像头**（Eye-in-Hand）。地上放了一块**AR 标定板**（Checkerboard）。机械臂动的时候，摄像头跟着动，每次都能"看到"标定板。

**问题**：摄像头是手工拧在末端上的，装歪几毫米、几度很正常。你不知道**摄像头相对末端的精确位姿**，导致：摄像头"看到"目标在某个位置，你让机械臂去抓，会偏。

**手眼标定**：用一系列已知运动，自动算出**末端到摄像头的变换 X**。

### 2. 涉及的坐标系

| 坐标系 | 符号 | 谁提供？ |
|--------|------|----------|
| 基座 Base | B | 机器人原点 |
| 末端 Hand | H | 正向运动学（已知） |
| 摄像头 Sensor | S | 装在末端上 |
| 标定板 Checkerboard | C | 世界中的固定参考 |

**核心关系**：$E_{bc} = E_{bh} \cdot X \cdot E_{sc}$

- $E_{bh}$：基座到末端（正向运动学，已知）
- $E_{sc}$：摄像头到标定板（摄像头检测 AR tag，已知）
- $E_{bc}$：基座到标定板（固定不变）
- **$X$**：末端到摄像头（**待求**）

### 3. 数学上在解什么

对任意两次测量 $i$、$j$，可以推出：$A X = X B$，其中 $A$、$B$ 由 $E_{bh}$、$E_{sc}$ 算出。

- **输入**：多组 $(A_i, B_i)$ 或等价的 $(E_{bh}, E_{sc})$ 测量
- **输出**：$X$（4×4 齐次变换：旋转 + 平移）
- **方法**：Park & Martin 的「先求旋转 $R_X$，再求平移 $t_X$」

### 4. 作业流程（663 完整版）

```
Day 1–3：在 Matlab 里实现算法
    ↓
generatedata：造「完美」合成数据，验证算法正确
    ↓
axxb_closedform：3 次测量 → 闭式解 X（精确）
    ↓
solveRx + solveTx + axxb：N 次测量 → 最小二乘解 X（抗噪声）
    ↓
Day 4：在 ROS 仿真里
    ↓
移动机械臂到 5+ 个位姿，用 tf2_echo 采集真实 E_bh、E_sc
    ↓
axxb(data.txt) → 得到 X
    ↓
在 Rviz 中对比：result 帧 vs 真实 camera 帧
    ↓
达标：平移误差 < 5 mm，旋转误差 < 0.1 rad → Expert Challenge 1pt
```

### 5. 663 的 Expert Challenge 要求

- 提交 `data.txt`：你采集的 gripper 和 marker 数据

### 6. 一句话总结

> **手眼标定**：利用「手」（末端）和「眼」（摄像头）在不同位姿下的测量，算出它们之间的固定变换 $X$，从而让摄像头能准确指导机械臂操作。

## 四天安排

### Day 1：环境 + 数据生成 + 理论复习

| 时段 | 任务 | 产出 |
|------|------|------|
| **上午** | 复习第 6–7 讲，重点 AX=XB 推导和 Park & Martin 方法 | 理清 A/B/X 与 E_bh/E_sc 的对应关系 |
| **下午** | 实现 `generatedata.m` | `randSE3()` + 循环生成 e_bh、e_sc |
| **晚上** | 验证 generatedata：用公式检查 E_bc = E_bh * X * E_sc | 能生成 N 组无噪声数据 |

**关键公式**：
- 方程：$E_{bc} = E_{bh} \cdot X \cdot E_{sc}$（常量）
- 对两帧 $i,j$：$A = E_{bh,i}^{-1} E_{bh,j}$，$B = E_{sc,i} E_{sc,j}^{-1}$，满足 $AX = XB$
- `randSE3()`：随机 R（可用 `orth(randn(3,3))` 或 Rodrigues），随机 t

**注意**：Matlab 四元数格式 `[qw qx qy qz]`，ROS 为 `[qx qy qz qw]`，输出时需转换。

### Day 2：闭式解 + solveRx

| 时段 | 任务 | 产出 |
|------|------|------|
| **上午** | 实现 `axxb_closedform.m` | 3 次测量 → 2 对 (A,B) → 闭式求 X |
| **下午** | 实现 `solveRx.m` | 极分解 $R_X = (M^T M)^{-1/2} M^T$，$M = \sum \beta_i \alpha_i^T$ |
| **晚上** | 用 generatedata 验证：闭式解和 solveRx 应精确还原 X | 误差 < 1e-10 |

**闭式解要点**：
- 3 次测量 → 取 (1,2) 和 (2,3) 得到 (A1,B1)、(A2,B2)
- $\alpha_i = \log(R_{A_i})$，$\beta_i = \log(R_{B_i})$
- $\mathcal{A} = [\alpha_1 \ \alpha_2 \ \alpha_1 \times \alpha_2]$，$\mathcal{B} = [\beta_1 \ \beta_2 \ \beta_1 \times \beta_2]$
- $R_X = \mathcal{A} \mathcal{B}^{-1}$

**solveRx 要点**：
- `logm` 对 SO(3) 不直接适用，需用第 7 讲的 `log_so3` 公式（或查 robotics toolbox）
- 极分解：`[U,S,V] = svd(M)` 则 $R = U V^T$（或按讲义用 $(M^T M)^{-1/2} M^T$）

### Day 3：solveTx + axxb + 自测

| 时段 | 任务 | 产出 |
|------|------|------|
| **上午** | 实现 `solveTx.m` | $(R_A - I) t_X = R_X t_B - t_A$，堆叠后最小二乘 |
| **下午** | 实现 `axxb.m` | 解析 e_bh/e_sc → 构造 A/B → 调用 solveRx、solveTx → 组装 X |
| **晚上** | 完整自测：generatedata(N) → axxb → 与真实 X 对比 | 所有函数通过合成数据验证 |

**solveTx 要点**：
- 对每组 $i$：$(R_{A_i} - I) t_X = R_X t_{B_i} - t_{A_i}$
- 堆成 $C \cdot t_X = d$，用 `C \ d` 或 `mldivide` 求解

**axxb 要点**：
- 从 e_bh、e_sc 每行提取 [tx ty tz qx qy qz qw]
- 构造 A_i、B_i（相邻帧或相对第一帧，按作业要求）
- 调用 solveRx(alphas, betas)、solveTx(RA, tA, RB, tB, RX)

## 常见坑

1. **四元数顺序**：`rotm2quat` 输出 [qw qx qy qz]，ROS 需要 [qx qy qz qw]，append 时重排
2. **axxb_closedform 输入**：作业写 3x7，应为 **3×7**（3 次测量，每次 7 个数），不是 3x7 矩阵的误写
3. **log_so3**：Matlab 的 `logm` 针对一般矩阵，SO(3) 需用轴角公式，可自写或查 robotics toolbox
4. **极分解**：`(M'*M)^(-1/2)` 用 `sqrtm` 或特征分解实现，注意数值稳定性
