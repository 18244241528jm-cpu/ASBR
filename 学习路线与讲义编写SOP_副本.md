### 阶段 0：按需补课（数学基础）

### 阶段 1：核心主线 A —— 刚体运动与运动学

| 顺序 | 讲义 | 对应 PPT | 核心内容 | 为什么在这个位置 |

| ② | **第 2 讲：2D/3D 刚体运动 — SO 与 SE 群** | lecture02 p1–11 | SO(2)、SE(2)、SO(3)、SE(3)、齐次坐标 | 所有运动学的数学语言 |
| ③ | **第 3 讲：前向运动学 — 从关节到末端** | lecture02 p12–20 | 坐标系链、齐次变换矩阵链乘、2D/3D FK | 正运动学是控制机器人的第一步 |
| ④ | **第 4 讲：逆运动学 — 从末端到关节** | lecture02 p21–23 | IK 问题定义、符号解 vs 数值解 | 正运动学的逆问题，实际应用必备 |

3. 翻对应 PPT 的图**再看一遍**，和文字对照

### 阶段 2：手眼标定

学完运动学后，自然的问题是：传感器装在机器人上，怎么知道传感器和机器人的相对位置？

| 顺序 | 讲义 | 对应 PPT / 论文 | 核心内容 |
|------|------|----------------|---------|
| ⑥ | **第 6 讲：手眼标定 — AX=XB 问题** | lecture03 p1–8 | Eye-in-Hand vs Eye-to-Hand、AX=XB 公式推导 |

## 第二部分：讲义编写 SOP（标准化处理流程）

### 流程总览

```
收到新 PPT
  │
  ▼
Step 1：浏览 & 拆分（10 分钟）
  │
  ▼
Step 2：按模板写每一讲（主要工作量）
  │
  ▼
Step 3：对照 Quality Checklist 逐项检查
  │
  ▼
Step 4：数值算例用代码验证
  │
  ▼
Step 5：更新学习路线（本文档）
```

---

### Step 1：浏览 & 拆分

1. **通读 PDF 全部页面**，记录每一页的标题和关键内容
2. 按**内容分界**（通常是大标题页或 "Recap → 新内容" 的转折）划分成 **2–5 讲**
3. 每讲对应 PPT 的明确**页码范围**
4. 写出每讲的**一句话目标**
5. 输出一个**学习大纲 md 文件**

---

### Step 2：按模板写每一讲

每讲的 Markdown 结构如下：

```markdown
# 第 X 讲：标题

**对应讲义**：xx.pdf 第 N–M 页
**本讲目标**：一句话

---

## 先想清楚：这讲到底在解什么问题？

用 3-5 句通俗话描述一个具体场景，说清楚"这讲到底在解什么现实问题"。
不涉及任何公式，让完全不懂的人也能读懂。

例："你有一台 UR5 机械臂，想让末端移动到桌上的杯子处。
你知道每个关节转了多少度，但怎么知道末端实际到了三维空间的哪个位置？
这就是前向运动学要解决的问题。"

---

## 和前面课程的关系

1-2 句或一个小表格，说明"上一讲学了什么 → 本讲为什么要学这个 → 下一讲去哪"。

---

## 1. 第一节标题

> 📖 **对照 PPT**：打开 xx.pdf **第 Y 页** "页面标题"，
> 描述图中的关键元素（2-3 句话说清楚图里画了什么）。

正文内容...

---

（中间穿插 🧭 过渡路标：在长推导段落之间、在"理论完 → 实际问题"的转折处）

**🧭 到这里你已经理解了 XX。下面进入 YY。**

---

## N. 数值算例：用具体数字走一遍

给定具体数值 → 逐步计算（写出每一步的数字）→ 用代码验证结果。
每讲至少一个算例。

```python
import numpy as np

# 验证代码
# ...
```

---

## N+1. 本讲小结

| 概念 | 要点 |
|------|------|
| ... | ... |

**学完本讲你应该能**：
1. ...
2. ...
3. ...

---

## N+2. 自测与练习建议

1. **口述**：不写公式，用自己的话向别人解释……
2. **手算**：给定具体数字……
3. **编程**：用 Python / NumPy……
4. **思考题**：……

完成以上后，可以进入 **第 X+1 讲：……**。
```

### Step 3：Quality Checklist

每讲写完后逐项检查：

| # | 检查项 | ✅ / ❌ |
|---|--------|---------|
| 1 | 有"先想清楚：这讲到底在解什么问题？"段落（通俗场景，无公式） | |
| 2 | 有"和前面课程的关系"段落（上一讲 → 本讲 → 下一讲） | |
| 3 | 至少 **3 处** `📖 对照 PPT` 标注（指明 PDF 文件名 + 页码 + 图中画了什么） | |
| 4 | 至少 **1 个** 数值算例（具体数字，逐步计算） | |
| 5 | 数值算例已用 **Python 代码验证**，数字准确 | |
| 6 | 首次出现的数学概念有**一句话直觉**解释（或"详见补充 X"指引） | |
| 7 | 长推导段落之间有 **🧭 过渡路标**（"到这里……下面……"） | |
| 8 | 自测中有"**口述**"题（检验理解深度，不写公式） | |
| 9 | 自测中有"**手算**"题（具体数字） | |
| 10 | 小结表格和"学完你应该能" checklist 齐全 | |

---

### Step 4：数值验证

所有讲义中出现的**具体数字**，必须用 Python 代码跑一遍确认正确。

验证方式：

```python
# 写一个简短脚本，复现讲义中的计算步骤
# 对比脚本输出和讲义中写的数字
# 如果不一致，以代码结果为准，修正讲义
```

> 为什么要这一步？因为手算/心算容易出错，一旦讲义里的数字错了，学的人会花大量时间怀疑自己算错了，浪费时间且打击信心。

---

### Step 5：更新学习路线

每次新增一组讲义后，回到本文档的"推荐学习路线"部分：

1. 把新讲义插入到正确的**阶段和顺序**中
2. 标注**前置知识**依赖
3. 如果新讲义需要前面没有的数学工具，考虑是否需要新增一篇**补充讲**

---

### 补充文档的分流策略

| 补充文档类型 | 处理方式 | 示例 |
|-------------|---------|------|
| **数学基础类** | 总结成独立"补充 M-X"讲义，在主线中加"详见补充 M-X"指引 | 线性代数、概率论 |
| **深度论文类** | 只总结**核心结论 + 与主线的对应关系**，不逐页翻译 | Park & Martin AX=XB 论文 |
| **工具/API 文档类** | 直接嵌入主线对应讲的"实战"节，以代码片段形式呈现 | ROS2 接口、UR5 驱动 |
| **作业/习题** | 不写单独讲义，在学习路线中标注为对应阶段的"检验" | HW1–HW5 |
