## 1. 本讲核心一句话

**边 (i, j) 存在** 当且仅当：从顶点 **qi** 到 **qj** 在关节空间 **逐点插值** 的整条路径上 **没有任何一点碰撞**。

两端点都无碰撞 **不够** —— 中间可能 **穿模**。

---

## 2. 关节空间距离（选边候选）

对两个顶点构型 **qa、qb**（同维数），常用 **欧氏距离**：

\[
d(q_a, q_b) = \sqrt{\sum_k (q_a^{(k)} - q_b^{(k)})^2}
\]

- **优点**：实现简单，与线性插值「走直线」在几何上自洽。  
- **注意**：全是 **弧度**；若有个别关节是 **平移轴**，要确认作业规划组是否只有转动关节（UR5 手臂通常为旋转关节）。

**与 Week05 PPT 的对应**：幻灯 **Distance Functions** 一节说，两构型之间的「距离」应 **启发式地** 反映 **连边是否容易成功**；**精确** 用扫掠体积等太难，**常用嵌入空间上的欧氏距离**（也可对平移/转动加权）。本讲采用的 **关节欧氏距离** 即此类 **heuristic**。**近邻搜索** PPT 提到 **kd-tree**、**格网哈希** 等以加速 kNN —— 作业中 **n 不大时暴力即可**；若需加速且环境允许，可再考虑 **FLANN** 等（须符合课程对依赖的要求）。

**复杂度**：若对每对 \((i,j)\) 都算距离并排序，朴素 **O(n²)**；n 在几千以上时 **make_edges** 可能成为瓶颈 —— 可先 **小 n 跑通**，再调参。

---

## 3. 连边策略：k 近邻 与 半径邻域

### 2.1 k 近邻（kNN）

对每个顶点 **i**，计算到所有 **j ≠ i** 的距离，取 **最近的 k 个** 作为候选边 **(i, j)**。

- **k 太小**：图易 **断连**。  
- **k 太大**：边数多、**碰撞检测次数** 暴涨，Plan 变慢。

**粗调**：k 从 **5～15** 试起，随场景变难增大。

### 2.2 半径邻域（r-disc）

连接所有满足 **d(qi, qj) < r** 的 **j**。  

- **r 太小**：断连；**r 太大**：边太多。  

可与 kNN **二选一**，或 **先半径再截断到最多 k 条**（工程里常见）。

### 2.3 无向图

PRM 通常 **无向**：若 **(i, j)** 合法，则 **(j, i)** 也合法。实现时可在 **邻接表** 里 **双向各加一条**，或 **只存 i < j** 但在最短路里当无向用 —— **与 `search_path` 约定一致即可**。

---

## 4. 边上碰撞检测：插值步长 Δ

**与 Week05 PPT 的对应**：幻灯 **Local Planner** 强调：局部规划器要 **快**；一种典型做法是在 **构型空间直线** 上运动，并对 **直线做细分** 做碰撞检测 —— PPT 举例 **二分递归**（不断对半分直到足够细）。你用的 **`interpolate` + 固定步长 Δ** 是 **等距离散**，与二分是 **同一思想的不同实现**（都是 **稠密检验** 直线段）。

对候选边 **(i, j)**：

```text
for s in { 0, Δ, 2Δ, … , 1 }   （包含 0 与 1，或按骨架 interpolate 约定）
    q ← interpolate(qi, qj, s)
    if state_collides(q):
        丢弃边 (i, j)，结束本边检验
若全程无碰撞:
    接受边 (i, j)，边权 w ← d(qi, qj)（或其它定义）
```

**Δ 的选取**：

| Δ 太大 | 中间细障碍 **漏检**，轨迹「穿墙」 |
|--------|-------------------------------------|
| Δ 太小 | 调用 **state_collides** 次数暴增，超时 |

**实践**：从 **0.05～0.1**（若 s 为归一化参数）量级试起；或固定 **最少采样点数**（例如每条边至少 **10～30** 个中间样本），再按最大关节变化缩放。

**端点**：若 `interpolate(..., 0)` 与 `(..., 1)` 已覆盖两端，且顶点已在 **make_vertices** 里保证无碰撞，理论上端点安全；仍建议 **与骨架的 interpolate 语义** 核对（是否包含端点）。

## 5. 邻接表与边权（给 `search_path` 用）

建议在类里维护例如：

```text
adj[i] = 若干 (j, weight) 的列表
```

- **weight**：常用 **d(qi, qj)**，则 **`search_path` 用 Dijkstra** 得到 **关节空间较短** 的路径。  
- **weight = 1**：退化为 **最少边数**，**BFS** 即可。

**自环**：不添加 **(i, i)**。  
**重复边**：同一 **(i, j)** 只保留一条（或取最小权）。

**每次规划前**：在 **`make_edges` 开头清空 `adj`**（或整图重建），避免 **上一次 Plan 的边** 残留。

---

## 6. 实现顺序建议（防一次写爆）

1. **暴力版**：对所有 **i < j** 算距离，暂时 **只连距离 < r** 的边，做插值碰撞 —— 验证 **能出合法边**。  
2. 改为 **kNN**，对比 **边数与连通性**。  
3. 调 **Δ、k、n** 直到简单场景 **Plan 成功率** 可接受。  
4. 再考虑 **FLANN** 等（须在 **ROS desktop 默认可用** 且 **CMake 声明依赖**；非必须）。

## 7. 伪代码：kNN + 边检验（语言无关）

```text
清空邻接表 adj[0..n-1]

for 每个顶点 i:
    建列表 candidates = 所有 j != i，带键 d(qi, qj)
    按距离排序，取前 k 个 j

for 每条无向候选边 {i, j}（注意去重：只处理 i < j 或用过集合判重）:
    if edge_valid(i, j):   // 内部：interpolate + state_collides
        w ← d(qi, qj)
        adj[i].append( (j, w) )
        adj[j].append( (i, w) )

edge_valid(i, j):
    for s from 0 to 1 step Δ:
        q ← interpolate(V[i], V[j], s)
        if state_collides(q): return false
    return true
```

**浮点步长**：用 **整数步数** 更稳，例如 `num_steps = ceil(1/Δ)`，对 `t = 0..num_steps` 令 `s = t / num_steps`。

## 8. 与 MoveIt / RobotState 的衔接

- **`interpolate`** 若为骨架提供，通常已基于 **同一 RobotModel**；传入的 **qi、qj** 应与 **V 中存储** 的类型一致。  
- 循环里可能对 **临时 RobotState** 赋值；注意 **不要** 在并行（若你以后扩展）里共用一个缓冲除非加锁 —— 本作业一般 **单线程**。

## 11. 常见误区

- **误区 1**：**双向邻接只加了一向** → 最短路只朝一个方向走。  
- **误区 2**：**Δ 过大** → RViz 里看到路径「穿过」薄板。  
- **误区 3**：**忘记清空 adj** → 第二次 Plan 用了 **旧图**。  
- **误区 4**：**边权用距离** 但 **search_path 当无权 BFS** → 仍可走通，但路径可能 **不够短**（非 bug，是设计选择；保持一致即可）。

## 12. 性能与专家挑战（预告）

Bugatti 等窄通道：**边变长**、**中间采样点增多**，单次 `make_edges` 可能接近 **评分时间上限**。可与 **`solve` 内重试**、**略增 Δ**、**限制 k** 等折中（见后续联调讲次）。
