## 1. `search_path` 到底搜什么？

- **输入**（典型）：**起点索引 vs**、**终点索引 vf**、以及类内已有的 **邻接表 `adj`**。  
- **输出**：若 **vs 与 vf 在同一连通分量**，返回一条 **顶点序列**（例如 `vs, …, vf`）；否则 **失败**。  

**不包含**（通常）：**start** 和 **goal** 一般 **不是** 图上的节点（除非你把它们也加进邻接表）。**与 Week05 PPT 的对应**：查询阶段在接上路网后要做 **图搜索**；PPT **Some Minor Issues** 还提到可对结果路径 **略作平滑（smooth）** —— 本作业 **不必实现** 平滑，若 MoveIt 或后续处理自带插值/时间参数化即可。

完整轨迹往往是：

```text
start  →  vs  →  …  →  vf  →  goal
        ↑ search_path 只负责中间这段 ↑
```

**两端悬边** 的离散点或轨迹段，在 **`solve`**（或辅助函数）里与中间段 **按顺序拼接**；以你的 **`assignment3_context.cpp`** 为准。

## 2. 边权与算法选择

| 边权 | 算法 | 说明 |
|------|------|------|
| 全为 **1**（只关心跳数） | **BFS** | 实现简单，O(V+E) |
| **w = d(qi, qj)**（关节距离） | **Dijkstra** | 非负权最短路；**推荐**与第 5 讲边权一致 |

**A\*** 需要启发式 **h(i)**，本作业 **非必须**；Dijkstra 足够。

**负权边**：你的碰撞检验是 **二值**（边要么存在要么不存在），权 **≥ 0**，**不会** 出现负权，无需 Bellman-Ford。

---

## 3. Dijkstra 要点（概念）

维护 **dist[i]** = 当前已知从 **vs** 到 **i** 的最短距离；**parent[i]** = 在最短路上 **i** 的前驱。

1. 初始化：`dist[vs] = 0`，其余 **∞**，**parent** 全为 -1。  
2. 用 **最小堆**（或 STL `priority_queue` 配合 `pair<dist, node>`）每次取 **dist 最小** 的未处理节点 **u**。  
3. 对 **u** 的每条邻边 **(u, v, w)**：若 `dist[u] + w < dist[v]`，则更新 **dist[v]**、**parent[v] = u**。  
4. 当 **vf** 被取出或确定最短路后，可 **提前结束**（可选优化）。  
5. 若结束时 **dist[vf]** 仍为 **∞**，则 **不可达**。

**复杂度**：约 **O((V + E) log V)**（堆实现）。

---

## 4. 路径重建（backtrack）

```text
若 dist[vf] 为无穷:
    失败

path ← 空列表
cur ← vf
while cur != -1:
    path.push_front(cur)   // 或先 push_back 再 reverse
    cur ← parent[cur]
若 path[0] != vs:
    数据异常，应检查
```

得到 **vertex_indices = [vs, …, vf]**。

---

## 5. BFS 何时够用？

若你把 **边权** 全部设为 **1**（每条边贡献相同），**BFS** 第一次到达 **vf** 的层数即 **最少边数**。  

**缺点**：关节空间实际代价与 **几何长度** 更一致；**加权 Dijkstra** 通常 **路径更短、更自然**。若作业评分 **不区分**，**BFS** 可接受；若希望 **更平滑**，用 **距离权 + Dijkstra**。

---

## 6. 与 `solve` 的衔接：从索引序列到轨迹

**search_path** 通常只返回 **整数索引列表**。完整运动需要：

1. **start → q[vs]**：第 6 讲的局部连接（或沿插值采样）。  
2. **q[vs] → q[u1] → … → q[vf]**：对 **相邻顶点** 在关节空间 **插值细分**（与 **Δ** 一致），或调用骨架提供的 **轨迹插补** 函数。  
3. **q[vf] → goal**：同上。

**若骨架** 接受 **waypoints 列表**（`RobotState` 序列），你按顺序 **压入**；若只接受 **一条 `RobotTrajectory`**，需按 **时间参数化** 把多段拼成 **一条**（具体 API 查 MoveIt 文档与课程代码）。

## 7. 伪代码：Dijkstra（语言无关）

```text
function dijkstra(vs, vf, adj):
    dist[0..n-1] ← ∞
    parent[0..n-1] ← -1
    dist[vs] ← 0
    Q ← 最小堆，压入 (0, vs)

    while Q 非空:
        (d, u) ← Q.pop_min()
        if d != dist[u]: continue   // 陈旧条目
        if u == vf: break           // 可选提前结束
        for (v, w) in adj[u]:
            if dist[u] + w < dist[v]:
                dist[v] ← dist[u] + w
                parent[v] ← u
                Q.push((dist[v], v))

    if dist[vf] 为 ∞: return 失败
    return reconstruct(parent, vs, vf)
```

## 10. 常见误区

- **误区 1**：**有向图** 只加 **u→v** 不加 **v→u**（若第 5 讲已双向则无此问题）。  
- **误区 2**：**path** 重建时 **顺序反了** → 机器人「倒走」关节序列。  
- **误区 3**：**vs == vf** 时未 **特判**（最短路径可为 **单点**）。  
- **误区 4**：**浮点权** 比较用 `==` 判相等 → 用 **epsilon** 或 **相对容差**。
