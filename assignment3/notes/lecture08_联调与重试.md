## 1. 为什么需要单独一讲「联调」？

PRM 是 **随机 + 图** 的算法：**同一场景** 多次 Plan 可能 **成功或失败**。评分环境用 **`colcon build`** 默认配置、**不特意开 Release 优化**（见 PDF）。你需要：

**与 Week05 PPT 的对应**：**Why Are Probabilistic Methods Effective?** 强调 **不显式构造** 整个自由空间，而依赖 **局部采样 + 简单连接 + 碰撞检测**；**概率完备性** 意味着 **时间足够 / 样本足够** 时，失败概率可压下去。**Some Minor Issues** 指出：图 **不连通** 可能是 **真不连通**，也可能是 **采样不够** —— 这正是本节 **重试、增大 n/k** 的理论旁证，而非盲目调参。

- 区分 **实现 bug**（必现错误）与 **概率性失败**（可重试缓解）；  
- 控制 **总耗时**（顶点采样 + 连边 + 最短路 + 轨迹填充）；  
- 用 **日志** 定位：卡在 **无顶点**、**无边**、**接不上 vs**、**vf 失败**、**图不连通** 还是 **轨迹接口失败**。

## 2. 建议在 `solve` 里加的「外层重试」

```text
for attempt = 1 .. R:   // R 如 2～5，视超时调整
    清空 V、E、邻接表等
    make_vertices()
    if |V| == 0: continue 或 直接失败
    make_edges()
    if search_accessibility 失败: continue
    if search_departability 失败: continue
    if search_path 失败: continue
    if 组装轨迹并交给 MoveIt 成功: return 成功
return 失败
```

**要点**：

- **每次重试** 应 **重新采样**（新随机序列）或 **至少** 保证 **图** 与 **随机性** 更新；否则重试无意义。  
- **R 过大** 且 **单次** 很慢 → 容易 **超过 30s**（专家挑战参考时间）。

## 3. 参数旋钮表（调参时只动这些，先记笔记再改代码）

| 符号 | 含义 | 调大通常效果 | 风险 |
|------|------|----------------|------|
| **n** | 顶点数 \|V\| | 图更密、连通性更好 | 采样 + **make_edges** 更慢 |
| **k** | k 近邻 | 边更多、连通性更好 | **make_edges** 边检验次数 ↑ |
| **Δ**（或步数） | 边上插值粒度 | 更不容易穿模 | 每条边 **state_collides** 次数 ↑ |
| **m** | accessibility/departability 尝试邻居数 | 更易接上 start/goal | 每次查询更慢 |
| **R** | 外层重试次数 | 成功率 ↑ | 总时间 ↑ |

**经验顺序**（简单场景先跑通）：先保证 **n、k 合理** → 再调 **Δ** 消除穿模 → 再调 **m** → 最后才 **猛拉 n**。

## 4. 失败类型 → 排查方向

| 现象 | 可能原因 | 先试 |
|------|-----------|------|
| **\|V\| 远小于 n** | 场景太挤、拒绝采样过高 | 降障碍难度；或检查 **关节范围** 是否写错 |
| **\|E\| ≈ 0** | k 太小、或距离太远 | 增大 **k** 或 **r** |
| **accessibility 失败** | **m** 太小或 **Δ** 穿模 | 增大 **m**；**减小 Δ**（更密） |
| **departability 失败** | goal 刁、或图在 goal 附近空 | **Update** 换目标；增大 **n** |
| **search_path 失败** | **vs、vf** 不连通 | **n、k**；或检查 **无向边** 是否双向 |
| **轨迹组装失败** | MoveIt 接口、时间参数化 | 查骨架 **成功返回** 前 **最后一步** |

**日志**：在 **每个阶段** 打一行 **INFO**（提交前按课程要求保留或删除）：`|V|`、`|E|`、**vs、vf**、**path 长度**、**attempt 编号**。

## 5. 时间与复杂度：心里要有数

- **make_edges**：若 kNN 对每点 **O(n log n)** 排序 + **O(k)** 条边 × **每条边 O(1/Δ) 次碰撞**，主导项常是 **边数 × 每边碰撞次数**。  
- **Dijkstra**：**O((V+E) log V)**，通常 **小于** 稠密图 **make_edges**（若 E 很大另说）。  
- **专家挑战**：窄通道需要 **更多顶点** 或 **更密局部图**，总时间易 **顶满** —— 需 **折中**（略增 Δ 步数换时间、或限制 **R**）。

## 9. 常见误区

- **误区 1**：把 **每次 Plan 失败** 都当成 **算法错** —— 先 **多试几次**、**换随机种子**。  
- **误区 2**：**日志** 打在 **高频内层循环**（每条边每个点）→ **终端卡死**、时间失真。  
- **误区 3**：**只优化 make_vertices** 的随机数，但 **邻接表** 未清空 → **旧边** 污染。  
- **误区 4**：**专家挑战** 才第一次测 **总时间** → 可能来不及改 **n/k**；应 **提前** 在简单场景测 **时间量级**。
