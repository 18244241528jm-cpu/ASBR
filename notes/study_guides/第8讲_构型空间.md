# 第 8 讲：构型空间 — 从工作空间到 C-space

**对应讲义**：potential.pdf 第 1–8 页
**本讲目标**：理解构型空间（Configuration Space）的概念，明白为什么运动规划在 C-space 中进行，以及障碍物如何映射到 C-space

---

## 先想清楚：这讲到底在解什么问题？

你有一台两关节的机械臂在桌上操作，桌上放着一个杯子（障碍物）。你想让机械臂从当前姿态移动到另一个姿态，但**不能撞到杯子**。

直觉上，你会想：在三维空间中找一条绕过杯子的路径。但问题是——机械臂不是一个"点"，它是一个有形状的连杆机构，每个连杆都可能撞上杯子。

**构型空间（C-space）** 的核心思想是：把整个机械臂"缩"成一个点，把障碍物"膨胀"到相应的大小，然后在一个**低维空间**中做"点"的路径规划。

## 1. 什么是构型（Configuration）？

> 📖 **对照 PPT**：打开 potential.pdf **第 2 页** "Configuration Space"，
> 提到了 C-space 概念最早由 Lozano-Pérez 在 1979 年提出。

**构型**（Configuration）是完整描述机器人姿态所需的**最少参数**。

| 机器人类型 | 构型 | C-space |
|-----------|------|---------|
| 平面上的点机器人（不旋转） | $(x, y)$ | $\mathbb{R}^2$ |
| 平面上可旋转的机器人 | $(x, y, \theta)$ | $\mathbb{R}^2 \times S^1$ = SE(2) |
| 两关节平面机械臂 | $(q_1, q_2)$ | $S^1 \times S^1$ = 二维环面（torus） |
| 6 轴工业臂 | $(q_1, \ldots, q_6)$ | $S^1 \times \cdots \times S^1$ |

**一句话直觉**：C-space 是"所有可能姿态的集合"，每个姿态是空间中的一个点。

## 3. 直观理解：两关节臂的 C-space

> 📖 **对照 PPT**：打开 potential.pdf **第 4–7 页**，
> 第 4 页画了工作空间中的两关节臂（左）和对应的 C-space（右，是一个正方形/环面）。
> 第 5–7 页展示了障碍物如何从工作空间映射到 C-space。

### 3.1 工作空间 vs 构型空间

**工作空间**（Workspace）：机器人实际活动的物理空间（你能看到的）。
- 两关节臂在一个平面上活动
- 障碍物是具体的物体

**构型空间**（C-space）：以关节角 $(q_1, q_2)$ 为坐标轴的抽象空间。
- 每个点代表一个完整的机械臂姿态
- 机器人变成了一个**点**

对于两关节臂：
- $q_1 \in [0°, 360°)$，$q_2 \in [0°, 360°)$
- C-space 是一个正方形（因为角度周期性，实际上是一个**环面/torus**——上下边相连、左右边也相连）

### 3.2 障碍物在 C-space 中的样子

> 📖 **对照 PPT**：potential.pdf **第 6–7 页**的图非常关键——左边是工作空间中的障碍物，
> 右边是该障碍物在 C-space 中的形状（一个复杂的弯曲区域）。

工作空间中一个简单的方块障碍物，在 C-space 中会变成一个**复杂的弯曲形状**。

原因：当机械臂的某个连杆碰到障碍物时，对应的 $(q_1, q_2)$ 组合就是 C-space 中的"禁区"。由于连杆是有长度的，碰撞区域的边界很复杂。

### 3.3 路径规划 = 在 C-space 中找路

在 C-space 中：
- **起点** A：当前关节角 $(q_1^{start}, q_2^{start})$
- **终点** B：目标关节角 $(q_1^{goal}, q_2^{goal})$
- **障碍区域**：C-space 中所有会导致碰撞的 $(q_1, q_2)$

**运动规划问题变成了**：在 C-space 中，从 A 点到 B 点，找一条**不经过障碍区域**的连续路径。

```
工作空间（复杂）                    构型空间（抽象但简洁）
┌──────────────┐               ┌──────────────┐
│  机械臂（有形状）│               │  q2           │
│  障碍物（方块）  │    ──映射→    │  ████  B      │
│  A         B  │               │  ████         │
│               │               │ A    ████     │
└──────────────┘               │       q1      │
                               └──────────────┘
  规划困难：要考虑整个臂            规划简单：机器人是一个点
  的每个部分会不会碰撞              只需要避开阴影区域
```

## 4. 数值算例：C-space 中的碰撞检测

**题目**：两关节臂，$L_1 = L_2 = 1$。工作空间中有一个圆形障碍物，圆心 $(1.2, 0.5)$，半径 $0.3$。判断 $(q_1, q_2) = (30°, 60°)$ 是否碰撞。

思路：检查连杆 1 和连杆 2 的任何一点是否进入了障碍物内部。

```python
import numpy as np

L1, L2 = 1.0, 1.0
obs_center = np.array([1.2, 0.5])
obs_radius = 0.3

def check_collision(q1_deg, q2_deg):
    q1, q2 = np.radians(q1_deg), np.radians(q2_deg)

    # 采样连杆上的点进行检测
    for s in np.linspace(0, 1, 20):
        # 连杆 1 上的点：从原点到关节2
        x1 = s * L1 * np.cos(q1)
        y1 = s * L1 * np.sin(q1)
        if np.sqrt((x1 - obs_center[0])**2 + (y1 - obs_center[1])**2) < obs_radius:
            return True

        # 连杆 2 上的点：从关节2到末端
        x2 = L1*np.cos(q1) + s*L2*np.cos(q1 + q2)
        y2 = L1*np.sin(q1) + s*L2*np.sin(q1 + q2)
        if np.sqrt((x2 - obs_center[0])**2 + (y2 - obs_center[1])**2) < obs_radius:
            return True
    return False

result = check_collision(30, 60)
print(f"(q1=30°, q2=60°) 碰撞? {result}")

# 扫描整个 C-space，找出所有碰撞构型
import matplotlib
matplotlib.use('Agg')  # 无显示环境
import matplotlib.pyplot as plt

q1_range = np.linspace(0, 360, 180)
q2_range = np.linspace(0, 360, 180)
collision_map = np.zeros((len(q2_range), len(q1_range)))

for i, q2 in enumerate(q2_range):
    for j, q1 in enumerate(q1_range):
        collision_map[i, j] = check_collision(q1, q2)

# collision_map 就是 C-space 中的障碍物区域
print(f"C-space 中碰撞区域占比: {collision_map.mean()*100:.1f}%")
```

## 5. C-space 的拓扑结构

> 📖 **对照 PPT**：potential.pdf **第 5 页**提到了环面的概念。

两个旋转关节的 C-space 不是一个普通的正方形——它是一个**环面**（torus）：

- $q_1 = 0°$ 和 $q_1 = 360°$ 是同一个构型 → 左右边"粘合"
- $q_2 = 0°$ 和 $q_2 = 360°$ 也是 → 上下边"粘合"

**实际影响**：路径可以从正方形的右边界"穿"到左边界（因为实际上是连续的），就像吃豆人游戏中从屏幕右边走到左边。

## 6. 本讲小结

| 概念 | 要点 |
|------|------|
| **构型** | 完整描述机器人姿态的最少参数 |
| **构型空间 (C-space)** | 所有构型的集合，机器人变成一个"点" |
| **C-obstacle** | 障碍物在 C-space 中的映射，形状通常很复杂 |
| **C-free** | C-space 中不碰撞的区域 |
| **运动规划** | 在 C-free 中找从起点到终点的连续路径 |
| **拓扑** | 旋转关节的 C-space 是环面，有"绕圈"的特性 |
