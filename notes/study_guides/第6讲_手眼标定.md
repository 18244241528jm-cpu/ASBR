## 先想清楚：这讲到底在解什么问题？

你把一个摄像头装在机械臂的末端（"Eye-in-Hand"），想用摄像头看到的画面来引导机械臂操作。

有个关键问题：**摄像头和机械臂末端之间的精确相对位置是什么？**

你装摄像头的时候是手工拧螺丝的，不可能做到完美对齐——差个几毫米、歪个几度。如果不知道这个偏差，摄像头"看到"目标在某个位置，你让机械臂去够，就会差之毫厘、谬以千里。

**手眼标定**就是用一系列已知运动来**自动计算出摄像头和末端之间的精确变换**。

## 1. 机器人上有很多坐标系

> 📖 **对照 PPT**：打开 lecture03.pdf **第 2 页** "Coordinate Frames"，
> 左图画了一个 6 轴机械臂，每个关节处都有坐标系；右图画了各种传感器（摄像头、力传感器等），每个传感器也有自己的坐标系。

一台配了传感器的机器人上，至少涉及这些坐标系：

| 坐标系 | 说明 |
|--------|------|
| **基座坐标系** (Base) | 机器人"脚"的位置，是全局参考 |
| **末端坐标系** (End-effector) | 机械臂最后一个关节的输出端 |
| **传感器坐标系** (Sensor/Camera) | 摄像头或其他传感器的光心位置 |
| **世界坐标系** (World) | 外部参考（比如标定板的位置） |

我们通过正向运动学知道基座和末端的关系（$E$）。传感器能测量自己和世界的关系（$S$）。但**末端和传感器之间的关系 $X$ 是未知的**——这就是要标定的东西。

## 2. 两种安装方式

> 📖 **对照 PPT**：打开 lecture03.pdf **第 3 页** "Combining Sensors and Robots"，
> 左图 "Eye-in-Hand"：摄像头装在机械臂末端，跟着末端一起动。

| 安装方式 | 特点 | 典型应用 |
|---------|------|---------|
| **Eye-in-Hand** | 传感器装在末端，跟着机器人动 | 精密装配、手术机器人 |
| **Eye-to-Hand** | 传感器固定在外部，观察机器人 | 工厂质检、物体识别 |

两种方式都需要标定，数学模型都归结为 **AX = XB** 方程。

## 4. AX = XB 方程推导

> 📖 **对照 PPT**：打开 lecture03.pdf **第 6–7 页** "AX=XB"，
> 第 6 页画了两个位置的机器人：位置 1（$E_1, S_1$）和位置 2（$E_2, S_2$），
> $X$ 是末端到传感器的未知变换，$B_1$ 是世界坐标系。

### 4.1 建立方程

机器人在**两个不同位置**采集数据：

**位置 1**：
- 正向运动学给出末端位姿 $E_1$
- 传感器测量到世界的变换 $S_1$

**位置 2**：
- 末端位姿 $E_2$
- 传感器测量 $S_2$

在两个位置，有一条**不变的链**（从基座到世界）：

$$E_1 \cdot X \cdot S_1 = E_2 \cdot X \cdot S_2$$

其中 $X$ 是末端到传感器的未知变换（不变，因为传感器是固定在末端上的）。

### 4.2 化简

$$X \cdot S_1 \cdot S_2^{-1} = E_1^{-1} \cdot E_2 \cdot X$$

定义：
- $A = E_1^{-1} E_2$：**机器人的相对运动**（从位置 1 到位置 2，末端怎么动了）
- $B = S_1 S_2^{-1}$：**传感器的相对运动**（传感器看到的相对位移）

得到核心方程：

$$\boxed{AX = XB}$$

**一句话解读**：
- $A$ 是**已知的**（正向运动学可算）
- $B$ 是**已知的**（传感器可测）
- $X$ 是**未知的**（要求的标定结果）

### 4.3 为什么叫"手眼"标定？

- "手" = 机器人末端（Hand），提供 $A$
- "眼" = 传感器/摄像头（Eye），提供 $B$
- 标定 = 找出"手"和"眼"之间的固定偏移 $X$

## 5. 一组数据够不够？

一组 $(A, B)$ 对应一个方程 $AX = XB$。$X$ 有 6 个自由度（3 旋转 + 3 平移），而一个方程提供的独立约束不够。

> PPT 第 8 页提到：有数十种方法求解 AX = XB，包括闭式解（四元数、轴角、对偶四元数）和迭代法（非线性最小二乘、凸优化）。

**结论**：至少需要**两组不同运动** $(A_1, B_1)$ 和 $(A_2, B_2)$，且旋转轴不平行，才能唯一确定 $X$。

实际中通常采集 **N >> 2 组**数据，用最小二乘法拟合最优的 $X$（因为测量有噪声）。

## 6. 数值算例：理解 A 和 B 的含义

**场景**：机器人从位置 1 移动到位置 2。

位置 1：$E_1 = \begin{pmatrix} I & (100, 0, 0)^T \\ 0 & 1 \end{pmatrix}$（末端在 x=100mm，无旋转）

位置 2：$E_2 = \begin{pmatrix} R_z(90°) & (0, 100, 0)^T \\ 0 & 1 \end{pmatrix}$（末端在 y=100mm，绕 z 转 90°）

则机器人的相对运动：

$$A = E_1^{-1} E_2$$

```python
import numpy as np

def make_transform(R, t):
    E = np.eye(4)
    E[:3, :3] = R
    E[:3, 3] = t
    return E

def Rz(deg):
    a = np.radians(deg)
    return np.array([[np.cos(a), -np.sin(a), 0],
                     [np.sin(a),  np.cos(a), 0],
                     [0,          0,         1]])

E1 = make_transform(np.eye(3), [100, 0, 0])
E2 = make_transform(Rz(90), [0, 100, 0])

A = np.linalg.inv(E1) @ E2
print("A = E1^{-1} @ E2 =")
print(np.round(A, 3))
# A 描述了末端从位置1到位置2的相对运动

# 如果已知 X（假设传感器相对末端平移了 (10, 0, 5)，无旋转）
X = make_transform(np.eye(3), [10, 0, 5])

# 则 B = X^{-1} A X （由 AX = XB 可得 B = X^{-1} A X）
B = np.linalg.inv(X) @ A @ X
print("\nB（传感器看到的相对运动）=")
print(np.round(B, 3))

# 验证 AX = XB
print("\nAX =")
print(np.round(A @ X, 3))
print("\nXB =")
print(np.round(X @ B, 3))
print("\nAX == XB?", np.allclose(A @ X, X @ B))
```

## 7. 本讲小结

| 概念 | 要点 |
|------|------|
| **手眼标定** | 求出传感器坐标系和机器人末端坐标系之间的固定变换 $X$ |
| **AX = XB** | $A$ = 机器人相对运动（FK 已知），$B$ = 传感器相对运动（测量已知），$X$ = 未知标定 |
| **Eye-in-Hand** | 传感器装在末端上 |
| **Eye-to-Hand** | 传感器固定在外部 |
| **数据需求** | 至少 2 组不同运动（旋转轴不平行），实际中用 N >> 2 组 |

3. 说明 $A, B, X$ 各自代表什么
4. 解释为什么一组数据不够，至少需要两组
